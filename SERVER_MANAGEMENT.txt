🤖 GROQ AI CHATBOT ИНТЕГРАЦИЯ
──────────────────────────────

Чатбот на сайте использует AI от Groq для интеллектуальных ответов.

УСТАНОВКА:
──────────
1. Установите Python SDK для Groq:
   pip install groq

2. API ключ уже задан в users/views.py (переменная GROQ_API_KEY)

КАК РАБОТАЕТ:
──────────────
- Пользователь пишет сообщение в чатбот (иконка 💬 справа внизу)
- JavaScript отправляет POST запрос на http://127.0.0.1:8000/chat/
- Django обрабатывает запрос в users/views.py view: chat_handler()
- Groq API (модель mixtral-8x7b-32768) генерирует ответ
- Ответ выводится в чатбот

МОДЕЛЬ AI:
──────────
- Модель: llama-3.3-70b-versatile (актуальная, поддерживаемая)
- Язык: Русский
- Контекст: Информация о ресторане Domi Sushi (интегрирована в system message)
- Макс токены: 500

ГОЛОСОВОЙ ВВОД:
────────────────
- Используется Web Speech API (встроенный в браузер)
- Поддерживаемые браузеры: Chrome, Firefox, Edge, Safari 14.1+
- Язык распознавания: Русский (ru-RU)

ПОЛУЧЕНИЕ НОВОЙ API КЛЮЧА:
────────────────────────────
1. Перейдите на https://console.groq.com/keys
2. Скопируйте ваш API ключ
3. Обновите значение GROQ_API_KEY в users/views.py строка ~25:
   GROQ_API_KEY = "ваш_новый_ключ"

⚠️ ВАЖНО: Храните API ключ в переменных окружения на production!
Для локальной разработки безопасно использовать в коде.

ТЕСТИРОВАНИЕ:
───────────────
1. Откройте http://127.0.0.1:8000/
2. Нажмите на иконку чатбота 💬 справа внизу
3. Напишите вопрос (например: "Какое меню у вас?")
4. Дождитесь ответа от AI

ОТЛАДКА:
─────────
- Проверьте API ключ в users/views.py
- Проверьте интернет соединение (требуется для запросов к Groq)
- В логах сервера должны быть сообщения типа "✅ Ответ от Groq: ..."
- Если groq не установлена, установите: pip install groq

ВОЗМОЖНЫЕ ОШИБКИ:
──────────────────
❌ "AI сервис недоступен"
   ✅ Установите groq: pip install groq

❌ "Ошибка при обработке запроса"
   ✅ Проверьте API ключ и интернет соединение

❌ "Таймаут при запросе"
   ✅ Сервер Groq может быть перегружен - повторите попытку

🎨 СИСТЕМА СВЕТЛОЙ И ТЕМНОЙ ТЕМЫ
───────────────────────────────────

Функционал:
───────────
- Кнопка переключения тем в навигации (справа)
- Иконка изменяется: 🌙 (светлая тема) ↔️ ☀️ (темная тема)
- Сохранение выбора в localStorage - тема сохраняется при перезагрузке
- Плавный переход 300ms при переключении
- Автоматическое определение системной темы (prefers-color-scheme)
- Работает на ВСЕХ страницах сайта

Светлая тема (по умолчанию):
────────────────────────────
- Фон: Белый (#ffffff)
- Текст: Темный (#1a1a1a)
- Карточки: Светло-серые (#f5f5f5)
- Иконка кнопки: 🌙 (луна)

Темная тема:
────────────
- Фон: Черный (#000)
- Текст: Белый (#fff)
- Карточки: Очень темные (#1a1a1a)
- Иконка кнопки: ☀️ (солнце)

Как использовать:
─────────────────
1. Откройте http://127.0.0.1:8000/
2. Найдите кнопку "Тема" с иконкой в навигации
3. Нажмите кнопку для переключения темы
4. Тема сохранится и будет применена при следующем визите

Технические детали:
───────────────────
- CSS переменные (root и data-theme):
  static/theme.css
  
- JavaScript управление:
  static/theme.js с классом ThemeManager
  
- Ранняя инициализация:
  Встроенные скрипты в <head> при загрузке (избегают мелькания)
  
- Файлы для отладки:
  Откройте в браузере: F12 → Console
  Там будут логи типа "🎨 ThemeManager инициализирована"
