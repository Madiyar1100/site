## 🎯 Как забронировать

### На главной странице (http://127.0.0.1:8000/):

1. **Нажмите кнопку "Брондау"** в верхней навигационной панели (золотая кнопка)
2. **Заполните форму**:
   - 📅 **Күні (Дата)** - дата бронирования
   - 🕐 **Уақыты (Время)** - время бронирования
   - 👥 **Адам саны** - количество человек (1-100)
   - 🪑 **Столик** - номер или описание столика (опционально)
   - 👤 **Аты-жөні** - ваше имя
   - 📞 **Телефон** - ваш номер телефона
   - ✉️ **Email** - ваша почта (опционально)
   - 💬 **Арнайы сілтеу** - специальные пожелания (опционально)

3. **Нажмите кнопку "✅ Брондау бақ"** для отправки

4. **Вы получите номер бронирования** в всплывающем окне

## 📊 Просмотр бронирований в админке

### Для администратора:

1. Откройте **http://127.0.0.1:8000/admin/**
2. **Логин**: `admin`
3. **Пароль**: `admin123`
4. Перейдите в раздел **"Бронирования"** (Users → Bookings)

### Там вы сможете:

✅ **Просмотреть все бронирования** с деталями:
- Имя клиента
- Дата и время
- Количество человек
- Контактная информация
- Статус

✅ **Редактировать бронирование**:
- Менять статус (Ожидание → Подтверждено → Выполнено/Отменено)
- Добавлять заметки администратора
- Изменять любые данные

✅ **Быстрые действия**:
- ✅ Подтвердить (mark_confirmed)
- ❌ Отменить (mark_cancelled)
- ✓ Выполнить (mark_completed)

✅ **Фильтровать и искать**:
- По статусу
- По дате
- По имени или телефону
- По тип бронирования

## 🔍 Проверка работы

### Тестовое бронирование создано:
- При запуске сервера создается тестовое бронирование (ID=1)
- Оно видно в админке в разделе "Бронирования"
- Можно редактировать, удалять или менять его статус

### Если бронирование не сохраняется:

1. **Проверьте логи сервера** (`Ctrl+Break` в терминале, а потом видны последние логи)
2. **Убедитесь что все поля заполнены** (имя, телефон, дата, время, количество человек)
3. **Проверьте консоль браузера** (F12 → Console) на наличие ошибок
4. **Перезагрузите страницу** (Ctrl+F5) и повторите попытку

## 📱 Формирование номера бронирования

- **ID бронирования** - это уникальный номер в системе
- Выводится в alert окне после успешного бронирования
- Используется для обращения в ресторан

**Все бронирования хранятся в базе данных SQLite (db.sqlite3) и полностью доступны для редактирования в админке!**
